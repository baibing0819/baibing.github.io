openSUSE Installation tutorial

First：

1、Insert the USB flash drive into the computer and boot it from the USB flash drive.
2、Select Install to enter the SUSE Installation Wizard
3、Select language—-english，Next

Nerwoek Setting：

Select Hostname/DNs : then modify your host name，Next

Systom Role:

Select KDE，Next

Partioning：

Select “start with existing partitioner” in the interface option “expert partitioner” to enter the partition interface. Since we currently have a window system, we choose to completely remove it: select sda, then select partitions, and delete them all.

Similarly, delete all your hard disk sdb.

Start partitioning your linux system：

Selectsda，Then Select Partitions–> add partition：

No. Partition name note size Rote format
1 / Create a system disk Operating System 40g ext4
2 /boot/efi Create a system boot partition EFI Boot Partition 0.5g FAT
3 swap Create a swap partition Swap 8g ext4
4 /soft Create a software disk Data and ISV Applications All remaining ext4

Add system disk, root directory image desc：

1、Select space size

2、Type is operating system

3、Select file format and file name

4、Create a software disk configuration:

The creation of other partitions is also the same step, just follow the form above to select the configuration.

Similarly, add partitions to sdb：

No. Partition name note size Rote format
1 /home Create a home directory partition Data and ISV Applications all ext4

Check that the entire partition is correct：

Confirmed without errors，click accept，next

Clock and Time Zone：

Region : Asia, Time ZOne:ShangHai , And correct the time

Modify the TIme：

Create user：

Software Installation：

need to install： C/C++ Development
Linux Kernel Development
Print Server
Technical Writing ->inkscape

fcitx google/cloud/sun (need search)

no need: game
office

Click ‘software’ to enter the software installation interface:

Search for fcitx to install Chinese input method if you need, you can select the googlepinyin ,cloudpinyin and sunpinyin

After completing the software selection , Start the installation ,It takes a while

SUSE configuration and installation of common software

First, add a software source：

1> Update software source
sudo zypper refresh

2>Add a software source, open the URL: https://en.opensuse.org/openSUSE:Science_Repositories,Select the Tumbleweed repository url and enter the following command in the terminal (the URL is  you just found):
 sudo zypper ar url/ science

3>Update the software source again
sudo zypper refresh

Select always：

Second, install NVIDIA driver

1> Search for related drivers

sudo zypper se nouv

2>Delete:

sudo zypper rm Mesa-dri-nouveau xf86-video-nouveau

3> Modify configuration file

sudo vim /etc/modprobe.d/50-blacklist.conf

Add at the end： blacklist nouveau

4>Make the configuration file take effect

sudo mkinitrd

5>reboot your linux

Third, the installation of Prime

1>add a software source
open the URL： SUSE_Prime,Add a software source and then update the software source sudo zypper refresh

2>Search for nvidia drivers

sudo zypper se nvidia

3>install

sudo zypper in nvidia-computrG05 X11-video-nvidiaG05

4>Follow the web page prompts to confirm that these files do not exist on your system.

5>安装suse-prime

sudo zypper in suse-prime

Switch cmd :

sudo prime-select nvidia

Fourth, install pymol

1>search and install pymol

sudo zypper se pymol

sudo zypper in python2-ptmol python3-pymol

2>Verify, enter pymol to open the software

Fifth, other tools

1> sudo zypper in python2-tk

2> sudo zypper se alsa
sudo zypper in alsa-devel

3> sudo zypper se yasm
sudo zypper in yasm yasm-devel

4> sudo zypper se pmw
sudo zypper in python2-pmw python3-pmw

Sixth, install mplayer

1>Log in to the cuda server, go to the /soft/source/mplayer folder, and copy the codecs folder to the local /usr/lib64, /usr/lib, /lib/win32, /lib64/win32 directories.

2>install amber
open the URL：http://ambermd.org/Installation.php
Copy the command to install the dependency

3>install the dependency
sudo zypper se gtk
sudo zypper in gtk2-devel python-gtk-devel

4>Unzip MPlayer-1.4.tar and go to the unzipped folder:
tar -vxf MPlayer-1.4.tar

5>To install in a graphical interface, enter the installation command:
./configure --enable-gui

6>Make the installation content take effect immediately:
sudo ldconfig

7>install htop ，Easy to view cpu running status:
sudo zypper in htop

8>Compile , note:this time you can open another terminal and enter htop to check the cpu operation.
make -j8

9>Install to local： sudo make install

10>Go back to the previous directory, extract and install the skin
tar -vxf clearlooks
sudo mv clearlooks /usr/local/share/mplayer/skin/default

11>Open mplayer to see if the installation is successful. Enter gmplayer

Seventh, install QT library and mplayer skin

1>sudo zypper se libqt4
``sudo zypper in libqt4-devel cmake

2> Install the skin of the player
tar -vxf smplayer××××××cd smplayer×××sudo make install``

4>Open smplayer to view the effect from the start menu search

eighth, Install mailbox

sudo zypper se thunderbird
``sudo zypper in Mozila thunderbird

Ninth, install wps

1>Copy the wps rpm package from the source in cuda to the local
2>install sudo rpm -ivh wpsxxxxx

Tenth, install the virtual machine (copy the vdi configuration file to the local)

1>sudo zypper se virtualbox
``sudo zypper in virtualbox virtualbox-guest-tools -

Choose the third:

2>Check the vbox group to use the virtual machine
1、Open yast, as shown in the figure:

2、Double click users

3、Select as shown in the figure, and then re-login the user after selecting

4、Open virtualbox again, we choose to create a new virtual machine

1> Click New

2>Fill in your virtual machine name and select it as shown below.

3>Choose to add a vdi file

Re-login your linux account

At this time, we have completed the establishment of the virtual machine. At this time, we open the newly created virtual machine and report the error. We need to follow the prompts to configure the terminal, as shown below:

Then execute the command again:
sudo /sbin/vboxconfig

After the completion, open the newly created virtual machine again and it will run normally.
